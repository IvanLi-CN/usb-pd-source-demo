# 淘宝采集任务规划（芯片级）

## 采集目标

- 目标芯片：`docs/candidates.md` 中每颗 SoC、升降压控制器与协议 IC（包含组合参考中的协议器件）。
- 每颗芯片完成一次淘宝采集并在原章节新增 `淘宝价格` 子章节，表格字段统一为：`型号`、`单价 (¥)`、`运费`、`店铺`、`链接`、`备注`。
- 单价按购买 1 件（或最小下单数量）计算；如为阶梯价或需选择规格，请在备注中写明条件。
- 运费以默认寄送地址显示为准；若包邮，填“包邮”。
- 备注需说明样品/代理、是否原厂、起订量、含税情况、券后价等关键限制。
- 每颗芯片最多收集 5 条有效信息；不足 5 条时写出“其余未找到”备注。
- 遇到单个宝贝包含多个符合条件的型号/规格（如颜色分类中列出 IP2366_BZ、IP2366_I2C 等），必须逐条展开记录在表格中；这些行合计视为 1 条有效信息。
- 若默认运费超过 10 元，视为无效报价并在备注写明排除原因。
- 仅记录裸片/芯片器件，套件、评估板、整机等不计入有效报价。

## Agent 行为规范

1. **搜索流程**
   - 使用 `chrome devtools`，访问淘宝搜索页。
   - 输入芯片型号或厂牌+型号（如 `IP2366`、`智融 SW6201`），避免附加“芯片”等额外关键词。
   - 仅浏览搜索结果前两页。
   - 对匹配项打开详情页确认规格、封装、功率或协议信息，确保与目标芯片一致。
2. **数据记录**
   - 摘抄所需字段，统一以人民币计价；如为阶梯价，以首档单件价为准并在备注说明。
   - 如果详情页无法确认具体型号或最小起订量，标记为“需确认”并酌情是否收录；严禁只写标题。

3. **上下文与效率**
   - 处理完一颗芯片即整理信息、更新 `docs/candidates.md` 对应章节的 `淘宝价格` 子章节。
   - 执行完一个芯片后应适当总结与压缩上下文，避免冗余。
4. **重复检验**
   - 若搜索结果全部为无关商品，需在任务记录中说明“未找到有效报价”。
