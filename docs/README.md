## 维护流程

1. **采集信息**
   - 通过淘宝首页检索目标型号，不添加“芯片”等额外词，逐页打开前两页结果。
   - 对合规报价记录型号、单价、运费、店铺、商品链接及备注（阶梯价、券后价等）。

2. **更新候选清单** (`candidates.md`)
   - 新增章节或补充“淘宝价格”子节。
   - 确保入围标准、特性、适用场景、性能指标和参考链接信息完整。
   - 在“对比表”同步加入或移除相应条目。

3. **维护黑名单** (`chip_blacklist.md`)
   - 若淘宝前两页无裸片报价或运费超标（>10 元），将型号加入表格并记下拉黑原因。
   - 一旦渠道放货，先从黑名单移除，再回补候选清单。

4. **提交前复核**
   - 检查 markdown 表格对齐。
   - 避免在候选清单和黑名单中出现同一型号的重复记录。
   - 简要总结更新内容后再进行 commit。

## 常见问题提示

- **缺少价格数据**：优先在黑名单记录说明，勿在 `candidates.md` 留空表格。
- **同 SKU 多规格**：应拆成多行，确保型号字段为实际裸片型号而不是商品标题。
- **引用来源**：所有外部规格或方案信息需附上可靠链接（充电头网、行业白皮书、官方渠道等）。
- **版本扩展**：如需添加新的文档（例如采集脚本、分析报告），请在本 README 中补充导航。
